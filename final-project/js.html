How Javascript was used to create this Website?

  1. The Home Page used a lot of click events particularly to redirect to other pages with eyecatching images or words that can be clicked on. Although they could have been done with hyperlinks I chose to do them with click events and location.href. The very first thing I created was the function fridayDeal() which was an if event that also used getDate() to check the day. If it was Friday then a special image would show that could be clicked on and redirect you to the orders page, otherwise it would just say "Order on Friday and everything is 20% off!". I couldn't think of much other content that a home page could have other than just news about the franchise or enticing deals. Click Here if you want to go to the Home Page.

  2. About Page wasn't going to use a lot of Javascript at first since it was just text, but I decided to put time into making an array and event listener for when the user clicks the image that changes the image AND the text below it at the same time. This didn't just use a normal array though, it used a MULTIDIMENSIONAL ARRAY which is one of the first things I challenged myself to try and learn. It was really cool to make work here since it made the JS look a lot more cleaner. This looks completely different from how my Location Page's JS looks. Click Here if you want to go to the About Page.

  3. Location Page was heavily focused on using geolocation. I tried using a multidimensional array to contain all of the locations, but couldn't figure it out. This was done before I made the "About Page". First the user's longitude and latitude is obtained then a function is run that gets the distance of multiple different locations, then calculates the distance from their current location in miles. This is the formula that I used for this function, but I had to change the radius of the Earth to transfer to miles instead of Kilometers. Click Here if you want to go to the Locations Page. https://stackoverflow.com/questions/18883601/function-to-calculate-distance-between-two-coordinates

  4. The Order Now Page had used a similar system to the D'oh Grill example which used a form checkbox and stored "values" that were the cost, and then you could "Submit" by evaluating the total with a button and output the result. There is also an option to "Empty Cart" which just resets the evaluated total so you can start fresh. Lastly I decided to follow up with the use of getDate() and the Friday checker I had already used and make it so that when it's Friday and you evaluate the total it applies a 20% coupon and gives feedback to the user that it was applied. Click Here if you want to go to the Order Page.

  5. The Contact and Reviews page was heavily towards form submission, alerts, and also using a script that got each line of information, concatenated it together, then displayed a NEW box in the reviews alongside the others. I was thinking of having the submitted review stay with localStorage, but I think it would be weird to "undo" it or unsubmit it since usually that's how reviews work, unless it was something like a "Delete Review" option instead. However, I did also have a function that checks the box "Good" and "Bad" that reads and applies the CSS coloring to the corresponding option. Click Here if you want to go to the Contact and Reviews Page.

  6. This is just a bonus, but on all pages there is a sign in option. This is a click event that when "clicked" opens a hidden form box. This uses localStorage so that when you fill it in and submit it the information will persist on page refresh and changing between pages on the site. The sign out option just resets it and makes the site forget you as if you just signed out. Lastly it's just a little thing but if you are signed in on the home page then it will give a personal message, and if you sign out it reverts back to normal.

Final Thoughts - I think I did good on the JS, but some parts can still look pretty ugly. I really don't like how the locations script looks even though it does work. I know it would have been nicer looking if I had gotten the Multidimensional Array to work, but maybe I gave up on it too soon. I think for certain the site would have been better looking if I spent more time on the CSS, but my thought process was that I wanted to put more work in the JS actually functioning and flexing out how much I could do with it first instead of visuals. I hope I did a good job with Website Responsiveness at the very least. Overall this was so much more fun to do than my CIS133 Website. I think because it was just being able to implement more interactable features instead of just having text, images, and different pages. I think for certain my most favorite stuff to work with was ids/query selectors, functions, if statements, events, geolocation, and maybe forms. Forms were hard to work with sometimes, but it was a fun challenge.
